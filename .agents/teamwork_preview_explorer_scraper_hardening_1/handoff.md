## 2. Logic Chain
1. **Invalid Impersonation Profiles**: `curl_cffi` depends on specific browser hello profiles compiled into its binary. Version names like `chrome146`, `safari2601`, and `firefox147` do not exist in current releases. Therefore, passing these values will throw a `ValueError` at runtime or fall back silently, rendering the TLS/JA3 bypass completely ineffective and exposing default libcurl signatures to Cloudflare.
2. **Hardcoded Headers Mismatch**: The HTTP headers in `STEALTH_PROFILES` are hardcoded. If they mismatch the TLS and HTTP/2 settings automatically generated by `curl_cffi` for the chosen browser version (e.g. sending standard Chrome client hint headers like `Sec-CH-UA` for Safari), WAFs will flag the discrepancy.
3. **No Automation Fallback**: `curl_cffi` cannot execute JS. If a website presents an interactive Javascript challenge (e.g. Cloudflare Turnstile, Datadome, PerimeterX), the fetch will fail. It must fall back to headless browser automation (`camoufox` or `nodriver`).
4. **Fragile BeautifulSoup Parsing**: The current `_parse_job_page` guesses tags (e.g. `h1`, class containing `"title"`, class containing `"company"`). Modern job sites standardise their data in Schema.org JSON-LD (`application/ld+json`). Parsing JSON-LD first provides 100% accurate structured data (list of dicts containing `title` and `url`), falling back to bs4 only when JSON-LD is missing.

## 3. Caveats
- Bypassing advanced anti-bot systems requires residential proxy rotation. If `RESIDENTIAL_PROXIES` is unconfigured, using free harvested proxies from `core/stealth.py` is safer than unproxied requests, but may still experience higher failure rates than premium residential IPs.
- We assume `curl_cffi`, `bs4`, `nodriver`, and `camoufox` are installed on the target environment.

---

## 4. Conclusion
The stealth scraper `scrapers/stealth_ingest.py` has critical vulnerabilities that will trigger WAF blocks on modern sites. Upgrading it to use valid `curl_cffi` profiles, dynamically align headers, reuse sessions/cookies, and fall back to browser automation on detection will resolve these blocks. Improving the parser to extract JSON-LD guarantees clean, structured parsed data. 

A complete proposed implementation has been created at `.agents/teamwork_preview_explorer_scraper_hardening_1/proposed_stealth_ingest.py`.
