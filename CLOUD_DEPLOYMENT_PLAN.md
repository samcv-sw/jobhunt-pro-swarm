# 🚀 خطة النشر السحابي المجاني (Zero-Cost Cloud Deployment Plan)

بناءً على طلبك لرفع النظام بالكامل على الـ Cloud بتكلفة **0$** وبشكل دائم، قمت بدراسة بيئة المشروع الحالية وتوفر منصات الاستضافة المجانية في عام 2026. 

توجد لديك العديد من الخطط ضمن المشروع (`DEPLOY_FREE.md`, `DEPLOY_0_COST.md`, `ORACLE_CLOUD_DEPLOYMENT.md`). سأقوم بدمجها وتطبيق أقوى خطة تضمن عدم توقف النظام أبداً (No Auto-Sleep).

---

## 🛠️ المعمارية المقترحة (Zero-Cost Architecture)

1. **الخادم الرئيسي (Backend & Engine):** Oracle Cloud Always Free (4 OCPUs, 24GB RAM).
    - *لماذا؟* لا ينام أبداً (Never Sleeps)، قوي جداً للتعامل مع متطلبات المتصفح المخفي (Camoufox) والذكاء الاصطناعي.
2. **قاعدة البيانات (Database):** Neon.tech PostgreSQL (الخطة المجانية).
    - *لماذا؟* قوية، تدعم الـ Connection Pooling، وتفصل بين التخزين والحوسبة.
3. **الواجهة الأمامية (Frontend Dashboard):** Vercel / Render.
    - *لماذا؟* مجاني مدى الحياة لاستضافة واجهات Vue.js و Node.js الخفيفة.
4. **اليقظة الأبدية (Keep-Alive):** GitHub Actions + Cron-job.org.
    - *لماذا؟* للتأكد من أن السيرفر لا يتوقف ولعمل نسخ احتياطي يومي لقاعدة البيانات.

---

## 📋 خطوات التنفيذ (Execution Steps)

أحتاج لموافقتك (Proceed) للبدء بتنفيذ الخطوات البرمجية التالية:

### الخطوة 1: تنظيف وتوحيد ملفات الإعداد (Docker & Config)
- سأقوم بإنشاء ملف `docker-compose.prod.yml` مخصص لبيئة Oracle Cloud يعمل بأقصى كفاءة بدون إهدار الموارد.
- سأقوم بتحديث `start_cloud.py` ليعمل كـ Entrypoint موحد يستدعي واجهة FastAPI (`web.app_v2`) بشكل صحيح.

### الخطوة 2: تحديث مسارات الواجهة الأمامية والوسيط (Frontend & Backend)
- المشروع يحتوي على مجلدات `backend-node` و `frontend-vue` ويبدو أنها مفصولة. سأقوم بإعداد ملفات `render.yaml` و `vercel.json` جاهزة للرفع مباشرة بضغطة زر.

### الخطوة 3: تجهيز سكربت التثبيت السحابي (Cloud Setup Script)
- سأقوم بكتابة سكربت متكامل `setup_oracle.sh` يمكنك تشغيله بضغطة واحدة داخل Oracle Cloud. هذا السكربت سيقوم بـ:
  1. تثبيت Docker و Docker Compose.
  2. سحب الكود من GitHub.
  3. تشغيل النظام عبر `docker-compose`.
  4. تجهيز الجدار الناري (UFW) لفتح بورت 80 و 443.

### الخطوة 4: حماية الـ API وربط قاعدة البيانات
- تجهيز متغيرات البيئة `.env.cloud` لاستخدامها مع **Neon PostgreSQL**.
